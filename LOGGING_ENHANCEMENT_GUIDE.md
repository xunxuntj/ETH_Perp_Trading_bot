## 🔍 如何使用日志验证指标正确性

### 场景 1：验证无信号的原因
当收到 `action="none"` 时，可以完整检查：
1. 检查 1H ST 方向是否与期望一致
2. 检查 1H 收盘价与 DEMA200 的大小关系
3. 检查 30m ST 方向
4. 根据这些数据判断为什么没有开仓信号

**示例：**
```
如果显示：
• 1H ST: 2040.25 🔴 红
• 1H 收盘: 2042.00 < DEMA 2044.50 ❌
• 30m ST: 2043.75 🟢 绿

→ 虽然 30m 是绿的，但 1H 是红的，所以不能开多
→ 这是正确的逻辑
```

### 场景 2：验证开仓信号的依据
当收到开仓信号时，能看到所有满足的条件：

```
• 1H ST: 2038.00 🟢 绿 ✅
• 1H 收盘: 2045.00 > DEMA 2040.00 ✅
• 30m ST: 2037.50 🟢 绿 ✅

→ 三个条件都满足，符合开多条件
```

### 场景 3：追踪持仓期间的指标变化
每次 `hold` 消息都显示当前的技术指标，可以逐周期观察变化：

```
周期 1: 
• 1H ST: 2008.50 🟢 绿
• 30m ST: 2009.00 🟢 绿

周期 2 (30分钟后):
• 1H ST: 2018.00 🟢 绿  (已变化)
• 30m ST: 2007.50 🔴 红  (已变化)

→ 能清晰看到 ST 的变化，推断是否即将触发平仓
```

## 🔍 故障排查指南

### 问题：总是收到 `action="none"`

**检查清单：**
```
1. 查看日志中的技术指标
2. 检查 1H ST 是否是期望的方向？
   - 若是🔴红，不能开多（只能开空）
   - 若是🟢绿，不能开空（只能开多）
3. 检查 1H 收盘与 DEMA200 的关系？
   - 多仓：需要 1H 收盘 > DEMA200
   - 空仓：需要 1H 收盘 < DEMA200
4. 检查 30m ST 方向是否与 1H 一致？
5. 三个条件都满足了吗？
```

### 问题：收到信号但觉得指标数据不对

**验证步骤：**
```
1. 在 GitHub Actions 日志中查看 details 部分
2. 与 Gate.io 实时行情对比
3. 确认时间戳是否是最新的
4. 检查是否使用了错误的 K-bar (应该是 iloc[-2])
```
