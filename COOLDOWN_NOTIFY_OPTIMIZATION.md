# 冷静期推送优化方案

## 问题描述

原本的冷静期推送存在问题：
- 脚本每30分钟运行一次
- 冷静期持续48小时
- 导致 48小时 ÷ 30分钟 = **96条重复推送** ❌ 过于频繁

## 解决方案

### 1. **智能推送策略**
- ✅ 冷静期**首次触发**时推送一次通知
- ✅ 之后的30分钟周期内**不再重复推送**
- ✅ 减少推送从 96条 → **1条** 🎯

### 2. **增加开单时间提示**
- 通知中明确显示**何时可以开单**
- 用户清楚地知道需要等待的具体时间

### 3. **自动状态管理**
- 使用 `cooldown_notify_state.json` 记录通知状态
- 冷静期结束后自动重置
- 下次进入冷静期重新推送

---

## 实现细节

### 状态文件结构

文件名: `cooldown_notify_state.json`

```json
{
  "notified": false,        // 是否已通知过
  "triggered_at": "2026-02-21T12:00:00+00:00",  // 冷静期触发时间
  "notify_count": 1         // 通知次数
}
```

### 工作流程

```
首次检查冷静期
  ↓
  ├─ 触发 (notified=false)
  │  ├─ 设置 notified=true
  │  ├─ 返回 should_notify=True
  │  ├─ strategy.py 返回 action="cooldown"
  │  └─ 推送1条通知 ✅
  │
  ├─ 30分钟后再次检查（仍在冷静期）
  │  ├─ 检查状态 (notified=true)
  │  ├─ 返回 should_notify=False
  │  ├─ strategy.py 返回 action="none"
  │  └─ 不推送 ✅
  │
  └─ 48小时后冷静期结束
     ├─ 重置状态 (notified=false)
     └─ 下次触发冷静期时重新推送
```

## 代码修改说明

### 1. cooldown.py 新增内容

```python
# 新增结构体字段
can_trade_time: Optional[datetime]  # 何时可以开单
should_notify: bool = False  # 是否应该推送通知

# 新增状态管理函数
load_cooldown_notify_state()      # 加载通知状态
save_cooldown_notify_state()      # 保存通知状态
reset_cooldown_notify_state()     # 重置通知状态（冷静期结束时）
```

### 2. strategy.py 修改内容

```python
# 原来：
if cooldown.triggered:
    return TradeResult(action="cooldown", ...)  # 总是推送

# 现在：
if cooldown.triggered:
    if cooldown.should_notify:
        action = "cooldown"  # 首次：推送通知
    else:
        action = "none"      # 非首次：不推送
    return TradeResult(action=action, ...)
```

### 3. main.py 无需修改

因为当 `action="none"` 时，不在 `notify_actions` 列表中，自动不推送。

## 技术细节

### 为什么用状态文件而不是数据库？

- ✅ 简单轻量，无需依赖
- ✅ 易于调试和手动修改
- ✅ 支持本地开发和GitHub Actions
- ✅ 支持容器化部署

### 冷静期结束后状态如何重置？

```python
# 在 check_cooldown() 中
if hours_since >= 48:  # 冷静期已过期
    reset_cooldown_notify_state()  # 自动重置
```

### 如果手动想重置状态怎么办？

```bash
# 删除状态文件
rm cooldown_notify_state.json

# 或者下次触发冷静期会自动创建新的
```

## 常见问题

### Q: 如果脚本中断，状态文件会丢失吗？
**A:** 不会。状态文件持久化存储在硬盘上。只要再次运行脚本，会继续读取状态。

### Q: 可以在冷静期中手动开单吗？
**A:** 不行。冷静期是强制的风控措施。状态文件只控制推送，不影响交易逻辑。交易执行由 `strategy.py` 的冷静期检查控制。

### Q: 如果想立即解除冷静期怎么办？
**A:** 
```bash
# 手动删除状态文件并修改平仓记录（不推荐）
rm cooldown_notify_state.json

# 或者，在config.py中临时改小MAX_CONSECUTIVE_LOSSES
MAX_CONSECUTIVE_LOSSES = 1000  # 临时改大，冷静期不会触发
```

### Q: 国际时区如何处理？
**A:** 所有时间都用UTC存储和推送。Telegram用户会看到UTC时间。建议根据自己所在时区加上时间差。
