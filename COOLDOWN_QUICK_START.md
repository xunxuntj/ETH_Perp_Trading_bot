## 🔍 工作原理

### 状态管理流程

```
运行脚本
  ↓
检查冷静期
  ↓
  ├─ 未触发冷静期
  │  └─ 正常交易信号处理
  │
  └─ 触发冷静期
     ├─ 读取通知状态
     │  ├─ notified=false (首次)
     │  │  ├─ 保存 notified=true
     │  │  ├─ 返回 should_notify=True
     │  │  └─ 🔔 推送1条通知
     │  │
     │  └─ notified=true (已通知)
     │     ├─ 返回 should_notify=False
     │     └─ 🤐 不推送（应用内显示）
     │
     └─ 冷静期结束（48小时后）
        ├─ 自动重置状态
        └─ 下次冷静期重新推送
```

### 核心代码变化

**cooldown.py** - 状态管理

```python
# 新增结构体字段
can_trade_time: Optional[datetime]  # 何时可以开单
should_notify: bool = False         # 是否应该推送通知

# 新增函数
load_cooldown_notify_state()    # 读取推送状态
save_cooldown_notify_state()    # 保存推送状态  
reset_cooldown_notify_state()   # 重置推送状态
```

**strategy.py** - 推送逻辑

```python
if cooldown.triggered:
    if cooldown.should_notify:
        action = "cooldown"  # ← 首次：推送通知
    else:
        action = "none"      # ← 已通知：不推送
```

## 📋 状态文件说明

### 文件位置
```
/workspaces/ETH_Perp_Trading_bot/cooldown_notify_state.json
```

### 文件内容示例
```json
{
  "notified": true,
  "triggered_at": "2026-02-21T12:00:00+00:00",
  "notify_count": 1
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|-----|-----|------|
| `notified` | bool | 是否已推送过冷静期通知 |
| `triggered_at` | string | 冷静期触发的时间 |
| `notify_count` | int | 推送次数 |

### 自动管理

- ✅ **自动创建**：首次进入冷静期时
- ✅ **自动更新**：冷静期首次触发时
- ✅ **自动重置**：冷静期结束时
- ✅ **持久存储**：关机也不会丢失

### 手动操作

```bash
# 查看当前状态
cat cooldown_notify_state.json

# 重置状态（如需要重新推送）
rm cooldown_notify_state.json

# 或通过Python
```

## 🧪 故障排除

### Q: 为什么没有收到冷静期通知？

**检查清单：**

1. 确认是否真的进入冷静期
   ```bash
   # 查看策略debug输出
   GATE_DEBUG=1 python main.py
   ```

2. 查看通知状态文件
   ```bash
   cat cooldown_notify_state.json
   ```
   如果 `notified=false`，说明还没推送过。

3. 检查Telegram配置
   ```bash
   # 确认环境变量设置正确
   env | grep TELEGRAM
   ```

### Q: 如何重新发送冷静期通知？

```bash
# 删除状态文件，重新运行脚本
rm cooldown_notify_state.json
python main.py
```

### Q: 状态文件可以手动编辑吗？

可以，但不建议。如需修改：

```json
{
  "notified": false,  // 改为false会重新推送
  "triggered_at": "...",
  "notify_count": 0
}
```
