## 🔍 技术细节

### 工作流程

```
运行脚本 (每30分钟)
  ↓
检查冷静期状态
  ├─ 未触发 → 正常处理
  └─ 触发冷静期
     ├─ 读取通知状态文件
     ├─ 检查是否已推送过
     │  ├─ 否（首次）→ 保存状态 + should_notify=True
     │  └─ 是（重复）→ should_notify=False
     └─ 返回结果
        ├─ should_notify=True → action="cooldown" → 🔔推送
        └─ should_notify=False → action="none" → 🤐不推送
```

## 💡 常见问题

**Q: 真的不会错过通知吗？**  
A: 不会。冷静期首次触发时一定会推送。之后的检查不推送是为了避免轰炸。

**Q: 如何确认开单时间？**  
A: 推送中明确显示："✅ 可开单时间: 2026-02-22 12:00 UTC"

**Q: 状态文件丢失会怎样？**  
A: 脚本会自动重建。下次进入冷静期时重新推送。

**Q: 能否手动重置？**  
A: 可以。删除 `cooldown_notify_state.json` 或调用 `reset_cooldown_notify_state()`。

**Q: 如何回滚到旧版本？**  
A: 删除新增文件即可。所有改动向后兼容。
